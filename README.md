#### 🧠 &nbsp;[Predictive Autoscaler for Kubernetes](https://github.com/Jignesh141)

Traditional Kubernetes HPA is reactive — it scales *after* things break. I built a system that predicts traffic spikes **before** they happen.

ML models (LSTM, Prophet) forecast CPU/memory with **91% accuracy** and trigger pod scaling **2–3 min before** load arrives. Integrated directly with the Kubernetes API for automated scaling, restarts, and intelligent alerting.

**↳** 40% faster incident response · 30% less resource waste vs HPA

#### 💬 &nbsp;[Distributed Real-Time Messaging](https://github.com/Jignesh141)

Designed for the problem most chat systems ignore — what happens when one server isn't enough.

**↳** 10K+ concurrent connections · Fault-tolerant delivery · Zero-downtime deploys
